# Firecrawl

Firecrawl is deployed as an internal-only service for Hermes website
extraction, scraping, and crawling support.

- API service: `http://firecrawl-api.firecrawl.svc.cluster.local:3002`
- Playwright service: `http://firecrawl-playwright.firecrawl.svc.cluster.local:3000`
- Redis, RabbitMQ, and nuq-postgres are ClusterIP-only support services.
- Only nuq-postgres is persistent, using `freenas-nvmeof-ssd-csi`.
- The optional upstream `nuq-prefetch-worker` is disabled. The verified
  self-hosted scrape path works through the API, worker, extract-worker, and
  nuq-worker; the prefetch process starts before queue tables exist in this
  deployment shape and remains unhealthy.

The manifests use the repo's `app-template` Helm pattern. The upstream
Firecrawl Helm chart in
`examples/kubernetes/firecrawl-helm` was used as the source model for process
commands, probes, services, and environment variables.

Secrets are generated by External Secrets Operator into `firecrawl-secret`.
No secret values are stored in git.

## Smoke Test

From a temporary in-cluster curl pod:

```bash
TOKEN="$(oc -n firecrawl get secret firecrawl-secret \
  -o jsonpath='{.data.TEST_API_KEY}' | base64 -d)"

oc -n firecrawl run firecrawl-smoke --rm -i --restart=Never \
  --image=curlimages/curl:8.18.0 -- \
  curl -fsS -H "Authorization: Bearer ${TOKEN}" \
  http://firecrawl-api:3002/v0/health/liveness
```

Hermes egress is intentionally not patched in this iteration. The follow-up is
to allow the Hermes namespace to reach `firecrawl-api.firecrawl.svc.cluster.local`
on TCP 3002 only.
